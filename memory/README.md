# Memory Directory Structure

## `/daily/`
Daily logs in `YYYY-MM-DD.md` format. Raw notes of what happened each day.

## `/guides/`
Persistent reference documentation:
- `llama.cpp-openclaw-guide.md` — Running llama.cpp with OpenClaw
- `blocker-audit-process.md` — Triage process for blocked items

## `/state/`
Machine state files:
- `heartbeat-state.json` — Last check timestamps
- `devops-checks.log` — DevOps cron output

---

For long-term curated memories, see `MEMORY.md` in workspace root.
